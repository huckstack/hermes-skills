## CORE LAWS

446. Distinguish between what you observed and what you inferred. Never treat an inference as a fact until confirmed.
447. Name the assumption you are about to act on before acting on it. If you cannot name it, you are not ready to act.
448. When you are wrong, say so immediately and completely. Partial acknowledgment wastes everyone's time.
449. The user watching your session has context you do not. They can see the screen, the logs, and the actual behavior. Treat their corrections as high-signal data, not interruptions.

## TECHNICAL SAFETY

2365. Read the file immediately before editing it. A mental model older than one edit is stale.
2366. Never use patch or str_replace for JavaScript files. Single quotes in JS break JSON serialization. Use sed or write_file.
2367. After any write_file, verify the change landed with wc -l or grep for a known string.
2368. Never patch a file more than 6 times in one session. After 6 patches, rewrite from scratch.
2369. When rewriting, write to a temp file first, validate it, then move it into place atomically.
2370. Always keep one backup before any destructive edit. One cp file file.bak is enough.
2371. Never assume indentation is correct after a patch. Re-read the patched section.
2372. When a file has grown beyond 1000 lines, split it before adding more features.
2373. Never edit the same line twice without re-reading the full function it belongs to.
2374. Specifying exact line numbers is safer than specifying code content when describing patch locations.
2375. When you are unsure what a file currently contains, read it. Do not guess.
2376. Partial writes are worse than no writes. Verify file integrity after every edit.
2377. Browser caching is aggressive and unreliable. Always append `?v=$(date +%s)` to URLs when reloading after file changes.
2378. Hard refresh does not always clear cache. Query string cache busting is more reliable.
2379. Always verify the server is serving the latest file with `curl -s localhost:PORT/file | grep "known_string"` before debugging browser behavior.
2380. Server process may still be serving old files after a disk write. Restart or check cache headers.
2381. When fetch returns correct data but the UI shows nothing, the JS is not executing. Check syntax first.
2382. Never navigate to a page and immediately check console in the same step. Wait for load completion.
2383. p5.js global mode variables are not accessible from browser console evaluation context. Do not try.
2384. CDN-loaded libraries can fail silently in sandboxed or offline contexts. Always confirm the library actually loaded before debugging your own code.
2385. Audio context initialization must be triggered by a user gesture in modern browsers. Never auto-initialize on page load.
